# MODULE_RULES ‚Äî thesis-structure (Screener swarm, Phase 1: M0.1‚ÄìM0.5)

These rules bind every agent in `.claude/agents/screener/thesis-structure/`. The root `CLAUDE.md` and `.claude/agents/screener/SWARM.md` apply in full; the stricter rule wins.

## What this module is

The first half of the Phase 1 assembly line. It turns a PROMOTED signal into the objective core of a thesis record: a sterile event statement (M0.1), quantified world changes (M0.2), a no-tickers beneficiary map (M0.3), a clock (M0.4), and a kill switch (M0.5). It is a hostile environment for weak ideas: raw input is systematically stripped of human bias, quantified, defended, and gated.

## The gates (all binding)

- **M0.1 ‚Äî causal-language gate.** The event statement contains who did what, when, where, and the immediate observable consequence ‚Äî and ZERO causal or interpretive language. Banned inside the statement: because, due to, driven by, as a result, leading to, signals, suggests, implies, panic, crisis, soaring, plunging, aggressively, inevitably ‚Äî and any synonym doing causal work. 2‚Äì4 sentences, minimum 50 characters. The check is performed and quoted (`causal_language_review` names the exact phrases checked).
- **M0.1 ‚Äî source confirmation gate.** The statement's facts are verified against an on-list source within the agent's first action. If the primary source reads clean (`primary_read_quality: full`), that direct read IS the confirmation. If the primary is paywalled or its body extraction is weak/failed, the agent runs a fallback search IN PRIORITY ORDER ‚Äî official company/IR/exchange ‚Üí regulator/filing ‚Üí Tier 1/2 media ‚Üí specialist finance blogs ‚Üí X/Twitter as weak corroboration only (`sources.thesis_structure.fallback_search_order`) ‚Äî logging every attempt, and feeds the resulting evidence packet (`source_grade`, `primary_read_quality`, the tier/confirms of each alternate found) to `scripts/screener_confirmation_score.py`. That script deterministically computes `confirmation_status` (confirmed / partially_confirmed / headline_only / unconfirmed), `extraction_confidence` (0‚Äì100, CLAUDE.md ¬ß12 bands), and the `gate_pass` boolean, which the agent copies verbatim into `60_second_source_check` ‚Äî never hand-asserted. A credible (Grade A) but totally uncorroborated headline is `headline_only`: it gets a real, non-zero confidence score reflecting that the source itself is credible, but it does NOT get a free pass ‚Äî the same numeric `gate_pass` threshold applies to everyone, so a `headline_only` record that doesn't clear it still routes `watchlist_no_source` (this is the deliberate, conservative design: it stays parked exactly as an unconfirmed signal would, but now fully labelled with what was tried and why, instead of an opaque fail). The instant ANY alternate source confirms ‚Äî even partially, even a weak X/Twitter signal ‚Äî confidence rises and the gate can open. A `human_prompt` signal whose facts cannot be matched to ANY source (primary or fallback) FAILS here ‚Üí `watchlist_no_source`. The agent never asserts a fact it did not actually confirm somewhere ‚Äî no hallucinating content from behind a paywall. `fallback_search_log` records every attempt for developer debugging and must never appear in human-facing prose (SWARM.md ¬ß8).
- **M0.2 ‚Äî reality lock.** Each world change states `quantitative_magnitude` (a number) against `baseline_reference` (the pre-event number) with `date_confirmed`. `already_occurred` is locked true; `hypothetical_flag` locked false. A change that has not already happened does not belong in the array ‚Äî it goes to `watchlist_deferred_items`. Minimum 2 items, maximum 6. Fewer than 2 quantified, already-occurred changes ‚Üí `gate_result: watchlist_no_world_change`.
- **M0.3 ‚Äî the 100-point impact matrix.** Each party (direct beneficiary / indirect beneficiary / harmed party) is an INDUSTRY or business model with a GICS classification ‚Äî never a company, never a ticker. Sub-scores, each 0‚Äì25: mechanism_directness (1-step = 25, 2-step = 15, 3-step/vague = 5), magnitude (quantified & large = 25, estimated = 15, directional only = 5), speed (impact within the M0.4 horizon = 25, lagged = 15, uncertain = 5), reversibility (hard to reverse = 25, partially reversible = 15, easily hedged/substituted = 5). Composite = sum. Tier: primary ‚â• 75 (carries to candidate-surfacing AND deep work), secondary 60‚Äì74 (carries), parked < 60 (logged only). The single invalid state: all three lists empty. Zero carry-forward ‚Üí `return_to_m0_2`.
- **M0.3 ‚Äî ticker check.** Machine-checkable: grep the draft for ticker-like tokens (uppercase 1‚Äì6 char symbols attached to exchange suffixes, $-prefixed cashtags, "NSE:"/"NYSE:" prefixes) and company names; record `ticker_check_detail {performed, violations, repair_action}`. Violations are repaired before saving, or the gate fails.
- **M0.4 ‚Äî observable expiry.** The horizon is one of: short_days_weeks / medium_weeks_3months / medium_long_3_6months / long_6months_plus. The expiry condition must be checkable tomorrow against a verifiable market signal (`expiry_condition_is_observable` locked true; `expiry_condition_is_opinion` locked false). "Sentiment improves" is an opinion, not an expiry.
- **M0.5 ‚Äî the kill switch.** One falsification sentence (‚â• 20 chars), specific enough to monitor tomorrow: two monitorable metrics, a numeric threshold with units, and a date by which crossing the threshold confirms falsification. The `uncomfortable_check` is locked true: the falsifier must genuinely threaten the thesis ‚Äî a soft, easily-dismissible condition fails the gate. Secondary falsifiers each carry a metric and a probability estimate. Once Phase 1 locks (edge-definition), these criteria CANNOT be moved.

## Routing (module synthesis)

- `Proceed` ‚Äî all gates pass ‚Üí edge-definition runs.
- `watchlist_no_world_change` ‚Äî M0.2 gate failed (terminal; record why and what was deferred).
- `return_to_m0_2` ‚Äî M0.3 produced zero carry-forward parties (terminal in this run; the record keeps the rationale so a human can revise and relaunch).
- (`watchlist_no_source` ‚Äî only via the M0.1 60-second check failing on a human_prompt.)

## Output discipline

- The synthesis assembles the DRAFT thesis record (meta + M0_1..M0_5) at `{RUN_ROOT}/thesis_record.json` with `locked: false`, `status: active` (or the terminal status), and `version: 1`. Field names follow `frameworks/screener/thesis_record.schema.json` exactly.
- Every agent report ends with a labelled `Verdict:` line; the synthesis ends with `## Machine Output` + `## Routing` (greppable `Routing:` line).
- ¬ß5 citations on every fact: `[Source, Date, Section]`. Every magnitude has a baseline. Banned without quantification: "could benefit", "may be impacted", bare "significant"/"material", "market hasn't realized".

## Writing Standard

SWARM.md ¬ß8 plain-English rules apply to every prose section in this module's output files. Machine-facing fields (JSON field names, M0.x codes, DIR-/IND-/HARM- party IDs, routing lines, `## Machine Output`) stay technical.

- **M0.1 event statement**: already has its own gate (sterile, no causal language) ‚Äî the gate language check does not conflict with ¬ß21; both apply. The statement itself stays sterile. The explanatory prose around the gate results must be plain English.
- **M0.2 world-change rows**: the table format stays. The "Deferred Items" rationale column: 1‚Äì2 plain sentences per row ‚Äî why this change is hypothetical, not yet confirmed.
- **M0.3 beneficiary map scoring notes**: 2‚Äì3 sentences per party explaining the score in plain English. Avoid academic phrasing like "exogenous demand-supply dynamics"; prefer "the event cuts supply, raising prices for everyone buying in the spot market."
- **M0.4 / M0.5**: these are short fields by design ‚Äî keep them crisp. The M0.5 falsification sentence must be plain enough to monitor without a finance background.
- **Synthesis Abstract**: 2‚Äì4 sentences. What the event was, what the key world changes are, what the blast radius looks like, and what the routing is. No M0.x codes in the abstract prose.
